WHY 35B-A3B IS DIFFERENT FROM 27B
───────────────────────────────────
Qwen3.5-35B-A3B is a Mixture-of-Experts (MoE) model.
  - Total parameters : 35B
  - Active per token : ~3B  (only a few experts fire per forward pass)
  - Memory footprint : similar to a ~7B dense model at fp16
  - Fits comfortably : on 2× H100 80 GB with tensor parallelism

Because only 3B params are active, the model is less sensitive to decoding
noise than a dense model.  The official Qwen eval config is:
  temperature  = 0.01  (NOT 0 — greedy can hurt MoE routing diversity)
  top_p        = 0.80
  top_k        = 20
  presence_penalty = 1.5

The custom eval scripts in h100/ detect the "-A3B" suffix automatically and
apply these settings — no manual changes needed.


================================================================================
SECTION 1 — LAUNCH SGLANG FOR 35B-A3B
================================================================================

On the vast.ai instance (or any 2×H100 machine):

  python -m sglang.launch_server \
    --model-path Qwen/Qwen3.5-35B-A3B \
    --tp 2 \
    --host 0.0.0.0 \
    --port 18000 \
    --reasoning-parser qwen3 \
    --tool-call-parser qwen3_coder \
    --mem-fraction-static 0.90

  Flags explained:
    --tp 2                    Tensor parallel over both GPUs (required)
    --reasoning-parser qwen3  Strips <think>…</think> server-side before
                              returning; the "content" field only has the answer
    --tool-call-parser qwen3_coder  Required for BFCL function-calling
    --mem-fraction-static 0.90      Use 90% of GPU memory for KV cache

  Wait for the server to be ready (~3-5 minutes for 35B-A3B):

    curl http://127.0.0.1:18000/v1/models | python3 -m json.tool

  Expected output:
    {
      "data": [
        { "id": "Qwen/Qwen3.5-35B-A3B", ... }
      ]
    }

  If the instance is remote, forward port 18000 to your local machine first:
    vastai ssh <INSTANCE_ID> -- -L 18000:localhost:18000

SECTION 2 — VERIFY THE MODEL AND CONFIG
================================================================================

Always run the diagnostic script before a full evaluation:

  cd /path/to/h100
  source venv/bin/activate   # or: pip install -r requirements.txt if first time

  python debug.py \
    --model Qwen/Qwen3.5-35B-A3B \
    --api-base http://127.0.0.1:18000/v1

  What to check in the output:
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 0. MODEL CONFIG                                                         │
  │    temperature: 0.01   ← MUST be 0.01, not 0 (MoE needs this)          │
  │    top_k      : 20     ← MUST be 20, not 1                              │
  │                                                                         │
  │ 1. SERVED MODELS                                                        │
  │    Qwen/Qwen3.5-35B-A3B  ← model name must match exactly               │
  │                                                                         │
  │ 2. RAW API RESPONSE                                                     │
  │    content: 'C'          ← no <think> leakage, clean single letter      │
  │                                                                         │
  │ 3. ANSWER EXTRACTION TEST                                               │
  │    extracted answer: 'C'  (expected 'C')  ✓                            │
  │                                                                         │
  │ 4. FIVE MMLU-PRO QUESTIONS                                              │
  │    Score: 5/5 | Failed extractions (None): 0/5                         │
  └─────────────────────────────────────────────────────────────────────────┘

  If temperature shows 0 or top_k shows 1:
    The model name was not recognised as MoE.  Check the exact spelling:
      Correct:   Qwen/Qwen3.5-35B-A3B
      Wrong:     Qwen/Qwen3.5-35b-a3b  (lowercase also works — auto-detected)
      Wrong:     Qwen3.5-35B-A3B       (missing "Qwen/" prefix is fine too)

  If <think> appears in content:
    SGLang was NOT started with --reasoning-parser qwen3.
    Kill the server and restart with that flag.

SECTION 7 — BFCL OFFICIAL FRAMEWORK (optional)
================================================================================

Use this section if you want to run the official BFCL leaderboard pipeline
in addition to the custom scripts.

7.1  Clone and install
────────────────────────
  git clone https://github.com/ShishirPatil/gorilla.git
  cd gorilla/berkeley-function-call-leaderboard
  pip install -e ".[all]"

7.2  Register Qwen3.5-35B-A3B in model_config.py
──────────────────────────────────────────────────
  Open:  bfcl_eval/constants/model_config.py
  Find the block where Qwen models are defined (search for "Qwen").
  Add these two entries after the existing Qwen entries:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  # Qwen3.5-35B-A3B (MoE — 35B total / 3B active)                       │
  │  "Qwen/Qwen3.5-35B-A3B-FC": ModelConfig(                               │
  │      model_name="Qwen/Qwen3.5-35B-A3B",                                │
  │      backend="sglang",                                                  │
  │      handler="QwenFCHandler",                                           │
  │      temperature=0.01,                                                  │
  │      top_p=0.8,                                                         │
  │      max_tokens=8192,                                                   │
  │  ),                                                                     │
  │  "Qwen/Qwen3.5-35B-A3B-Prompt": ModelConfig(                           │
  │      model_name="Qwen/Qwen3.5-35B-A3B",                                │
  │      backend="sglang",                                                  │
  │      handler="PromptingHandler",                                        │
  │      temperature=0.01,                                                  │
  │      top_p=0.8,                                                         │
  │      max_tokens=8192,                                                   │
  │  ),                                                                     │
  └─────────────────────────────────────────────────────────────────────────┘

  Key differences from the 27B dense entries:
    temperature=0.01   (not 0.001 — MoE models benefit from light sampling)
    handler stays QwenFCHandler  (same architecture family)

7.3  Register in supported_models.py
──────────────────────────────────────
  Open:  bfcl_eval/constants/supported_models.py
  Add these two strings to the list of supported model names:

    "Qwen/Qwen3.5-35B-A3B-FC",
    "Qwen/Qwen3.5-35B-A3B-Prompt",

7.4  Set environment variables
────────────────────────────────
  export LOCAL_SERVER_ENDPOINT=http://127.0.0.1
  export LOCAL_SERVER_PORT=18000

  Confirm SGLang is already running with Qwen3.5-35B-A3B before continuing.

7.5  Run the official BFCL pipeline
─────────────────────────────────────
  # Step 1: generate model outputs
  python -m bfcl generate \
    --model "Qwen/Qwen3.5-35B-A3B-FC" \
    --test-category web_search \
    --num-gpus 2 \
    --backend sglang \
    --skip-server-setup \
    --temperature 0.01

  # Step 2: score the outputs
  python -m bfcl evaluate \
    --model "Qwen/Qwen3.5-35B-A3B-FC" \
    --test-category web_search

  --skip-server-setup tells BFCL not to launch its own SGLang instance
  (you already have one running at port 18000).

================================================================================
SECTION 9 — TROUBLESHOOTING
================================================================================

Problem: temperature shows 0 in debug output (wrong for MoE)
  Fix: verify the model name contains "-A3B" (case-insensitive check in code)
       python -c "from eval.api_client import get_config; print(get_config('Qwen/Qwen3.5-35B-A3B'))"
       Should print: temperature=0.01

Problem: <think> blocks appear in content field
  Fix: restart SGLang with --reasoning-parser qwen3
       pkill -f sglang.launch_server
       python -m sglang.launch_server --model-path Qwen/Qwen3.5-35B-A3B \
         --tp 2 --port 18000 --reasoning-parser qwen3 \
         --tool-call-parser qwen3_coder --mem-fraction-static 0.90 &

Problem: MMLU-Pro / MMMU scores are ~50% (near random chance)
  Cause: temperature is too high or model is not loading correctly.
  Fix A: run debug.py to confirm temperature and extraction.
  Fix B: check that --model flag matches exactly what /v1/models returns.
         curl http://127.0.0.1:18000/v1/models

Problem: BFCL null_predictions > 20/100
  Cause: model not returning {"answer": "..."} JSON format.
  Fix A: check that --reasoning-parser qwen3 is active (think blocks cause
         the JSON to be buried inside <think>...</think>).
  Fix B: increase MAX_TURNS in eval/bfcl_websearch.py (default 15).

Problem: Video-MME scores very low or zero
  Cause: decord not installed or video files not found.
  Fix A: pip install decord av
  Fix B: verify videos downloaded: ls ./data/videomme/videos/ | head -5

Problem: DuckDuckGo rate-limit errors during BFCL
  Cause: too many parallel workers hitting DDG API.
  Fix: reduce workers to 2: --workers 2
       The script already has exponential backoff but heavy parallel load
       can exhaust retries.

Problem: OOM (out of memory) on GPU
  Cause: --mem-fraction-static too high or model shards not fitting.
  Fix: reduce to 0.85: --mem-fraction-static 0.85
       35B-A3B needs ~28 GB across both GPUs at bf16 for weights alone.


================================================================================
SECTION 10 — COMPARING 27B vs 35B-A3B RESULTS
================================================================================

After running both models, compare using the summary JSONs:

  python3 - <<'EOF'
  import json

  for model_tag in ["Qwen_Qwen3.5-27B", "Qwen_Qwen3.5-35B-A3B"]:
      path = f"results/summary_{model_tag}.json"
      try:
          d = json.load(open(path))
          print(f"\n{d['model']}")
          for bench, score in d["scores"].items():
              tgt = d["targets"].get(bench)
              if tgt:
                  delta = score * 100 - tgt
                  print(f"  {bench:<22} {score*100:.2f}%  (target {tgt:.1f}%,  delta {delta:+.2f}%)")
              else:
                  print(f"  {bench:<22} {score*100:.2f}%")
      except FileNotFoundError:
          print(f"  [{path} not found — run that model first]")
  EOF
